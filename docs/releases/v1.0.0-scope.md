# HyperGit `v1.0.0` Scope Freeze

本文冻结 `v1.0.0` 对外承诺范围，用来约束 CLI、文档、帮助文本和测试口径。

## In Scope

- 本地工作区命令：`init`、`status`、`add`、`commit`、`log`、`diff`、`checkout`、`hydrate`、`dehydrate`、`sparse add/remove`、`doctor`。
- 远端命令：`fetch` / `push`，且只承诺 `file://` remote。
- 存储与索引：canonical codec v1、loose object、segment pack、commit graph、workspace state / local change / sparse profile 持久化。

## Out Of Scope

- `merge` CLI：底层 merge 模块保留，但 `v1.0.0` 不承诺用户可见命令、冲突流程或 shell 测试。
- `branch` CLI：`v1.0.0` 不承诺分支管理命令。
- `clone` CLI：`v1.0.0` 不承诺单独的 `hgx clone` 用户命令；当前推荐流程是 `hgx init` 后执行 `hgx fetch file://...`。
- HTTP remote：当前只保留 smoke / 评估级实现，不进入 `v1.0.0` 发布承诺。
- Git 互操作：当前仍是 prototype / proof of concept，不进入 `v1.0.0` 承诺矩阵。
- `policy_id` / `dedupe_scope` / `audit event`：当前仍是实验性默认元数据，不提供发布级策略执行、审计流水或跨仓去重承诺。

## Remote Baseline

- `v1.0.0` 发布级远端主路径固定为 `file://` remote。
- `fetch` / `push` 必须继续通过端到端 happy path、CAS failure 和 sparse fetch 覆盖。
- HTTP remote 后续若进入正式范围，将在 `v1.1+` 单独补协议验证、鉴权和对象批量传输能力。
