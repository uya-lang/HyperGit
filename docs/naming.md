我会叫它：

# **HyperGit**

含义很直接：

* **Hyper**：面向超大规模、超越传统 Git 的边界
* **Git**：保留 Git 的核心精神：内容寻址、不可变对象、分布式协作
* 听起来像基础设施级系统，不像玩具项目

完整定位可以是：

> **HyperGit：A Parallel Version Control System for Planet-Scale Repositories**

中文名可以叫：

> **极仓**

寓意：

* 极大规模仓库
* 极快并行处理
* 极致版本管理

所以品牌组合可以是：

```text
HyperGit / 极仓
```

命令行工具可以叫：

```bash
hgx
```

避免和 Mercurial 的 `hg` 冲突，同时 `x` 表示 extended、extreme、experimental。

例如：

```bash
hgx clone repo
hgx status
hgx diff
hgx sparse add src/payment
hgx hydrate src/core/**
hgx affected tests
```

我最推荐：

> **HyperGit（极仓）**

技术感、可传播性和定位都比较强。
